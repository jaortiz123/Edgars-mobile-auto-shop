## 3) Manual Review Checklists (mark ‚úÖ/‚ùå + notes)

### 3.1 State Coverage
- ‚òê **Loading**: skeletons where helpful; spinners < 300ms only if needed.
- ‚òê **Empty**: clear explanation + primary CTA; no ‚Äúblank abyss.‚Äù
- ‚òê **Success**: visible confirmation (toast/badge/redirect) with undo when safe.
- ‚òê **Error**: friendly message, retry, error code, correlation ID logged.
- ‚òê **Permission‚ÄëDenied (403)**: explain who to contact; no silent redirect.
- ‚òê **Offline**: cached shell + reconnect banner; queue mutations or disable safely.

### 3.2 Forms
- ‚òê Labels tied via `htmlFor`; `aria-describedby` points to error text.
- ‚òê Client + server validation with identical rules/messages.
- ‚òê Submit disabled while pending; no double‚Äësubmission.
- ‚òê Preserve entered data on error; focus moves to first invalid field.
- ‚òê Error copy tells the user what to do next.

### 3.3 Navigation & Routing
- ‚òê Deep links work; refresh safe.
- ‚òê `404` and `403` pages exist.
- ‚òê Protected routes gate and preserve intended destination (`returnTo`).

### 3.4 Responsiveness & Layout
- ‚òê Breakpoints cover xs/sm/md/lg/xl; no horizontal scroll on mobile.
- ‚òê Tap targets ‚â• 44√ó44px; hit areas padded; sticky footers don‚Äôt hide content.
- ‚òê `meta viewport` correct; respects iOS safe area; keyboard does not block inputs.

### 3.5 Accessibility (WCAG 2.2 AA)
- ‚òê Color contrast ‚â• 4.5:1 text; ‚â• 3:1 for large text/icons.
- ‚òê Focus **visible** everywhere; logical tab order; no traps.
- ‚òê Semantic landmarks: `main`, `nav`, `header`, `footer`.
- ‚òê Headings hierarchical (one `<h1>` per screen).
- ‚òê Images have descriptive `alt`; decorative use empty alt.
- ‚òê Live updates use `aria-live` (`polite` for toast/info, `assertive` for critical).
- ‚òê Motion respects `prefers-reduced-motion`.

### 3.6 Content & i18n
- ‚òê No hard‚Äëcoded user‚Äëvisible strings (use i18n).
- ‚òê Dates/numbers localized; pluralization correct.
- ‚òê RTL layouts render correctly (smoke test at least primary pages).

### 3.7 Telemetry & Debuggability
- ‚òê UX events named consistently (`ui.<area>.<action>`).
- ‚òê Error boundaries log to Sentry with user/tenant context and trace ID.
- ‚òê Feature flags drive experimental UI; flags are server‚Äëauthoritative.

## 8) Risk Scoring & Prioritization
- **Critical:** Missing error boundaries on async pages; no state coverage for critical flows; inaccessible forms preventing submission.
- **High:** Poor mobile layout; unreadable contrast; broken keyboard nav; uncontrolled toasts (not announced).
- **Medium:** Inconsistent empty states; confusing copy.
- **Low:** Cosmetic inconsistencies, minor spacing/typography drift.

Prioritize by **Risk (Likelihood√óImpact)** then **Effort (S/M/L)**.
